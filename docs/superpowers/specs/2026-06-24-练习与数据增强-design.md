## 1. 背景与目标

当前产品闭环：**录入（标注词库）→ 挖空练习 → SM-2 间隔复习 → 浏览器扩展高亮**。
本次增强目标：

1. 让练习更顺手（键盘流）、更有学习反馈（拼写提示）。
2. 把已沉淀但未露出的数据可视化（仪表盘），并新增**词级**易错统计。
3. 打通"网上读到 → 收藏 → 练习"的闭环（扩展存句）。

非目标（YAGNI，本次不做）：用户鉴权/多用户隔离、单词级 SM-2、AI 自动标注、导出/导入、PWA。

## 2. 数据层改造

### 2.1 新表 `BlankAttempt`（词级易错榜的数据来源）

原因：现有 `PracticeRecord` 只记录句级（答对几个空 / 共几个空），无法还原"哪个词错了"。词级统计需按空记录，并**快照保存当时的单词文本**，使历史统计不受后续改/删句子影响。

```sql
create table "BlankAttempt" (
  id text primary key,
  "sentenceId" text,
  "articleId" text,
  word text,            -- 该空英文答案，归一化（lower + 压空格）快照
  hint text,            -- 中文释义快照
  "isCorrect" boolean,  -- 按"首次作答"判定
  revealed boolean,     -- 该次是否走了"显示答案"
  "createdAt" timestamptz default now()
);
-- 易错榜查询性能
create index "BlankAttempt_word_idx" on "BlankAttempt" (word);
create index "BlankAttempt_createdAt_idx" on "BlankAttempt" ("createdAt");
```

RLS：与现有表保持一致（沿用项目既有策略；本项目无鉴权，依赖 anon key + 既有 RLS 现状）。

> 决策记录：备选方案是给 `PracticeRecord` 加 `blanks jsonb` 列、前端聚合。已选**新建表**：聚合查询干净、可加索引、随历史增长不拖慢、对句子编辑健壮。

### 2.2 其余指标零改动

正确率趋势、活跃热力图、KPI、掌握度分布均可由现有 `PracticeRecord` + `ReviewState` 计算，无需改表。

## 3. 功能设计

### 3.1 #6 键盘流（练习页 PracticePage）

- 进入新句：**自动聚焦第一个空**（`autoFocus` 或对第一个 input 调 `ref.focus()`，在切句 effect 后执行）。
- `Tab` / `Shift+Tab`：在空格间移动（原生即可；确保 input 间无干扰元素抢焦点）。
- `Enter`：未检查时 = 检查；已检查时 = 下一句/查看结果。
- 与自动跳题不冲突：全对自动跳题后，新句的焦点回到第一个空。

文件：`src/pages/PracticePage.tsx`。

### 3.2 #5 拼写提示（练习页，方案 A：提示+重试，按首次判定）

状态机（单句）：
1. **未检查**：正常填空。
2. **第一次「检查」**：
   - 计算每个空的首次对错 `firstResults: boolean[]`。
   - **立即落库（且只此一次）**：`recordPractice` + `upsertReview` + 新增 `recordBlankAttempts`，全部使用首次结果。
   - 全对 → 成功 toast + 自动跳题（保留现有 900ms 延时与竞态修复，见 3.5）。
   - 有错 → 进入**重试态**：错的空显示 **首字母 + 长度占位**（例 `probe` → 提示 `p····`，5 个字符），**不锁定输入**，对的空保持只读且标绿。
3. **重试态再「检查」**：揭示完整答案 + 标红/绿 → 显示「下一句」。**不再落库**（首次已记）。
4. 「显示答案」按钮：任意时刻可点，揭示答案、记为未掌握（`revealed=true`）；若尚未落库则此刻按 `revealed` 落库一次。

提示渲染：占位串由 `answer[0] + '·'.repeat(answer.length-1)`（多词答案按整串长度，空格保留）。作为 input 的 `placeholder` 或下方灰字提示，二选一在实现时定（倾向 placeholder，避免布局跳动）。

"错"的定义（供易错榜/SM-2）：**首次作答错即记为错**，无论重试是否答对。提示只帮记忆，不改判定。

新增落库函数：`recordBlankAttempts(sentenceId, articleId, blanks[], firstResults[], revealed)` → 批量 insert `BlankAttempt`，每空一行，`word`/`hint` 取自 `parseContent` 的标注快照（归一化 word）。

文件：`src/pages/PracticePage.tsx`、`src/services/practiceService.ts`（加 `recordBlankAttempts`）。

### 3.3 #2 学习仪表盘（新「数据」Tab）

入口：`App.tsx` 增加第 4 个 Tab「数据」，挂载新页面 `src/pages/StatsPage.tsx`。

布局（方向 C + 词级易错榜）：
- **顶部 KPI 卡 ×4**：累计练习句 · 已掌握 · 今日待复习 · 连续打卡天。
- **正确率趋势**（近 14 天，按天：当天 `isCorrect` 句数 / 当天总句数）。
- **掌握度分布**（按 `ReviewState.reps` 分桶：0 / 1 / 2 / 3+）。
- **活跃热力图**（近 6 周，按本地自然日的 `PracticeRecord` 计数，分 4 档色阶）。
- **最易错词 Top N（词级）**：来自 `BlankAttempt`，`group by word` 数 `isCorrect=false`；点一项 → 跳「练习」并用包含该词的句子集开练（复用 `startCustomPractice`）。

指标口径（可调，写入实现常量）：
- **已掌握**：`ReviewState.reps >= 2`。
- **今日待复习**：`ReviewState.nextReviewAt < 明日 0 点`（与 `fetchReviewItems` 口径一致）。
- **连续打卡天**：从今天（或昨天，若今天还没练）起，本地自然日连续有 ≥1 条 `PracticeRecord` 的天数。
- **累计练习句**：`PracticeRecord` 去重 `sentenceId` 数（或总记录数，二选一，spec 取"去重句数"）。

数据获取：新增 `src/services/statsService.ts`，封装：
- `fetchPracticeRecords()`：拉 `PracticeRecord`（含 `isCorrect, sentenceId, createdAt`）。
- `fetchReviewStatesRaw()`：拉 `ReviewState`（reps, nextReviewAt）。
- `fetchTopErrorWords(limit)`：拉 `BlankAttempt` 聚合（前端聚合即可；量大后可改 Supabase RPC）。
- 纯计算（趋势/热力图/连击/分布）抽到 `src/lib/stats.ts`，**纯函数、可单测**。

性能：个人应用数据量小，客户端拉取 + 聚合可接受；若 `PracticeRecord` 显著增长，后续可加 SQL 聚合/RPC（本次不做）。

文件：`src/App.tsx`、`src/pages/StatsPage.tsx`（新）、`src/services/statsService.ts`（新）、`src/lib/stats.ts`（新）。

### 3.4 #3 扩展闭环：智能预标注（方案 3）

交互：
1. 网页选中一句英文 → 选区旁浮出**「存入 WordBread」**小按钮（content script 监听 `selectionchange`/`mouseup`）。
2. 点按钮 → 弹出轻量面板（注入的 shadow DOM 或定位 div，复用扩展现有样式风格）：
   - 显示整句；**自动把句中已在 `wordMap` 的词标为生词并带中文**（复用 background 已建的 `wordMap`）。
   - 用户可点词增/删生词；为新词补中文释义（行内输入）。
   - 选择目标文章：默认"网页收藏"（首次自动创建），可下拉改选（从 Supabase 拉 Article 列表）。
3. 「存入」→ content 拼成 `(word-中文) ...` 标注格式，POST `/rest/v1/Sentence`（与 background 读取同一套 `WB_CONFIG.url/key`）。
4. 成功提示；失败保留面板并提示重试。

实现要点：
- 复用 `src/lib/annotation.ts` 的标注格式约定（写出与解析对称）。扩展侧已有 `extension/lib/annotation.js`（`extractAnnotations`），需补一个"按 wordMap 在纯文本里包裹标注"的逆向函数 `buildAnnotatedContent(text, wordMap)`。
- "网页收藏"文章：存句前确保存在——查 Article（title='网页收藏'）无则 POST 创建，拿到 id。
- 写库经由扩展（background 或新 `lib/api.js`）统一封装 `createSentence` / `ensureInboxArticle` / `fetchArticles`。
- content script 面板：纯原生 DOM，隔离样式（shadow DOM 优先），避免污染宿主页面。

文件：`extension/content.js`（选区按钮 + 面板）、`extension/background.js` 或新 `extension/lib/api.js`（写库封装）、`extension/lib/annotation.js`（加 `buildAnnotatedContent`）、`extension/manifest.json`（如需权限/资源声明）。

### 3.5 既有自动跳题竞态修复（顺带）

当前 `setTimeout(handleNext, 900)` 未清理，用户在 900ms 内手动跳题/切文章会跳两次。改造：用 `useRef` 存 timer，在切句 effect 与组件卸载、以及任何手动跳题/显示答案时 `clearTimeout`。纳入 #5/#6 同批改动。

## 5. 错误处理

- 所有 Supabase 写入沿用现有风格：失败 `console.error` + 返回 false / toast 提示，不阻断练习。
- `recordBlankAttempts` 失败不影响 `recordPractice`/`upsertReview`（各自独立，互不阻断）。
- 仪表盘任一数据源失败：该卡片显示空态/占位，不整页崩溃。
- 扩展写库失败：面板保留 + 错误提示，允许重试；网络/权限错误打印到扩展控制台。
- 扩展"网页收藏"文章创建失败：提示并中止本次存句。
